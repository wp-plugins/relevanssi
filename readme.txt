    <table class="code"><thead><tr><th class="lineno" title="Line numbers">Line</th><th class="content"> </th></tr></thead><tbody><tr><th id="L1"><a href="#L1">1</a></th><td>=== Relevanssi ===</td></tr><tr><th id="L2"><a href="#L2">2</a></th><td>Contributors: msaari</td></tr><tr><th id="L3"><a href="#L3">3</a></th><td>Donate link: http://www.mikkosaari.fi/relevanssi/</td></tr><tr><th id="L4"><a href="#L4">4</a></th><td>Tags: search, relevance</td></tr><tr><th id="L5"><a href="#L5">5</a></th><td>Requires at least: 2.6.5</td></tr><tr><th id="L6"><a href="#L6">6</a></th><td>Tested up to: 2.8.1</td></tr><tr><th id="L7"><a href="#L7">7</a></th><td>Stable tag: 1.4</td></tr><tr><th id="L8"><a href="#L8">8</a></th><td></td></tr><tr><th id="L9"><a href="#L9">9</a></th><td>Relevanssi replaces the basic WordPress search with a partial-match search that sorts the results based on relevance.</td></tr><tr><th id="L10"><a href="#L10">10</a></th><td></td></tr><tr><th id="L11"><a href="#L11">11</a></th><td>== Description ==</td></tr><tr><th id="L12"><a href="#L12">12</a></th><td></td></tr><tr><th id="L13"><a href="#L13">13</a></th><td>Relevanssi replaces the basic WordPress search with a partial-match search that sorts the results</td></tr><tr><th id="L14"><a href="#L14">14</a></th><td>based on relevance. It is a partial match search, so if user inputs several search terms, the</td></tr><tr><th id="L15"><a href="#L15">15</a></th><td>search will find all documents that match even one term, ranking highest those documents that match</td></tr><tr><th id="L16"><a href="#L16">16</a></th><td>all search terms.</td></tr><tr><th id="L17"><a href="#L17">17</a></th><td></td></tr><tr><th id="L18"><a href="#L18">18</a></th><td>Relevanssi does some fuzzy matching too, so if user searches for something that doesn't produce</td></tr><tr><th id="L19"><a href="#L19">19</a></th><td>any results, Relevanssi will look for similar terms.</td></tr><tr><th id="L20"><a href="#L20">20</a></th><td></td></tr><tr><th id="L21"><a href="#L21">21</a></th><td>The matching is based on basic tf * idf weighing, with some extra features added like a boost for</td></tr><tr><th id="L22"><a href="#L22">22</a></th><td>words that appear in titles.</td></tr><tr><th id="L23"><a href="#L23">23</a></th><td></td></tr><tr><th id="L24"><a href="#L24">24</a></th><td>In general the plugin doesn't affect the display of search results at all - that is left for the </td></tr><tr><th id="L25"><a href="#L25">25</a></th><td>search result template to decide. However, if the option is set, Relevanssi will create custom</td></tr><tr><th id="L26"><a href="#L26">26</a></th><td>search result snippets that show the part of the document where the search hit was made. Relevanssi</td></tr><tr><th id="L27"><a href="#L27">27</a></th><td>can also highlight the query terms in the search results.</td></tr><tr><th id="L28"><a href="#L28">28</a></th><td></td></tr><tr><th id="L29"><a href="#L29">29</a></th><td>Relevanssi can keep a log of user queries and display both most popular queries and recent queries</td></tr><tr><th id="L30"><a href="#L30">30</a></th><td>that got no hits. The logging is a new feature that will be refined later.</td></tr><tr><th id="L31"><a href="#L31">31</a></th><td></td></tr><tr><th id="L32"><a href="#L32">32</a></th><td>Relevanssi supports the hidden input field `cat` to restrict searches to certain categories (or</td></tr><tr><th id="L33"><a href="#L33">33</a></th><td>tags, since those are pretty much the same). Just add a hidden input field named `cat` in your</td></tr><tr><th id="L34"><a href="#L34">34</a></th><td>search form and list the desired category or tag IDs in the `value` field. You can also set the</td></tr><tr><th id="L35"><a href="#L35">35</a></th><td>description from general plugin settings (and then override it in individual search forms with</td></tr><tr><th id="L36"><a href="#L36">36</a></th><td>the special field).</td></tr><tr><th id="L37"><a href="#L37">37</a></th><td></td></tr><tr><th id="L38"><a href="#L38">38</a></th><td>Relevanssi owes a lot to [wpSearch](http://wordpress.org/extend/plugins/wpsearch/) by Kenny</td></tr><tr><th id="L39"><a href="#L39">39</a></th><td>Katzgrau.</td></tr><tr><th id="L40"><a href="#L40">40</a></th><td></td></tr><tr><th id="L41"><a href="#L41">41</a></th><td>The plugin might work with WordPress versions prior to 2.6.5 - that's just oldest release I've</td></tr><tr><th id="L42"><a href="#L42">42</a></th><td>tried. I do know the plugin uses a function introduced in 2.5.</td></tr><tr><th id="L43"><a href="#L43">43</a></th><td></td></tr><tr><th id="L44"><a href="#L44">44</a></th><td>== Installation ==</td></tr><tr><th id="L45"><a href="#L45">45</a></th><td></td></tr><tr><th id="L46"><a href="#L46">46</a></th><td>1. Extract all files from the ZIP file, and then upload the plugin's folder to /wp-content/plugins/.</td></tr><tr><th id="L47"><a href="#L47">47</a></th><td>1. If your blog is in English, skip to the next step. If your blog is in other language, rename the file *stopwords* in the plugin directory as something else or remove it. If there is *stopwords.yourlanguage*, rename it to *stopwords*.</td></tr><tr><th id="L48"><a href="#L48">48</a></th><td>1. Activate the plugin through the 'Plugins' menu in WordPress.</td></tr><tr><th id="L49"><a href="#L49">49</a></th><td>1. Go to the plugin settings and build the index following the instructions there.</td></tr><tr><th id="L50"><a href="#L50">50</a></th><td></td></tr><tr><th id="L51"><a href="#L51">51</a></th><td>To update your installation, simply overwrite the old files with the new, activate the new</td></tr><tr><th id="L52"><a href="#L52">52</a></th><td>version and if the new version has changes in the indexing, rebuild the index.</td></tr><tr><th id="L53"><a href="#L53">53</a></th><td></td></tr><tr><th id="L54"><a href="#L54">54</a></th><td>If you want to use the custom search results, make sure your search results template uses </td></tr><tr><th id="L55"><a href="#L55">55</a></th><td>`the_excerpt()` to display the entries, because the plugin creates the custom snippet by replacing</td></tr><tr><th id="L56"><a href="#L56">56</a></th><td>the post excerpt.</td></tr><tr><th id="L57"><a href="#L57">57</a></th><td></td></tr><tr><th id="L58"><a href="#L58">58</a></th><td>If you're using a plugin that affects excerpts (like Advanced Excerpt), you may run into some</td></tr><tr><th id="L59"><a href="#L59">59</a></th><td>problems. For those cases, I've included the function `relevanssi_the_excerpt()`, which you can</td></tr><tr><th id="L60"><a href="#L60">60</a></th><td>use instead of `the_excerpt()`. It prints out the excerpt, but doesn't apply `wp_trim_excerpt()`</td></tr><tr><th id="L61"><a href="#L61">61</a></th><td>filters (it does apply `the_content()`, `the_excerpt()`, and `get_the_excerpt()` filters).</td></tr><tr><th id="L62"><a href="#L62">62</a></th><td></td></tr><tr><th id="L63"><a href="#L63">63</a></th><td>To avoid trouble, use the function like this:</td></tr><tr><th id="L64"><a href="#L64">64</a></th><td></td></tr><tr><th id="L65"><a href="#L65">65</a></th><td>`&lt;?php if (function_exists('relevanssi_the_excerpt')) { relevanssi_the_excerpt(); }; ?&gt;`</td></tr><tr><th id="L66"><a href="#L66">66</a></th><td></td></tr><tr><th id="L67"><a href="#L67">67</a></th><td>== Frequently Asked Questions ==</td></tr><tr><th id="L68"><a href="#L68">68</a></th><td></td></tr><tr><th id="L69"><a href="#L69">69</a></th><td>= What is tf * idf weighing? =</td></tr><tr><th id="L70"><a href="#L70">70</a></th><td></td></tr><tr><th id="L71"><a href="#L71">71</a></th><td>It's the basic weighing scheme used in information retrieval. Tf stands for *term frequency*</td></tr><tr><th id="L72"><a href="#L72">72</a></th><td>while idf is *inverted document frequency*. Term frequency is simply the number of times the term</td></tr><tr><th id="L73"><a href="#L73">73</a></th><td>appears in a document, while document frequency is the number of documents in the database where</td></tr><tr><th id="L74"><a href="#L74">74</a></th><td>the term appears.</td></tr><tr><th id="L75"><a href="#L75">75</a></th><td></td></tr><tr><th id="L76"><a href="#L76">76</a></th><td>Thus, the weight of the word for a document increases the more often it appears in the document and</td></tr><tr><th id="L77"><a href="#L77">77</a></th><td>the less often it appears in other documents.</td></tr><tr><th id="L78"><a href="#L78">78</a></th><td></td></tr><tr><th id="L79"><a href="#L79">79</a></th><td>= What are stop words? =</td></tr><tr><th id="L80"><a href="#L80">80</a></th><td></td></tr><tr><th id="L81"><a href="#L81">81</a></th><td>Each document database is full of useless words. All the little words that appear in just about</td></tr><tr><th id="L82"><a href="#L82">82</a></th><td>every document are completely useless for information retrieval purposes. Basically, their</td></tr><tr><th id="L83"><a href="#L83">83</a></th><td>inverted document frequency is really low, so they never have much power in matching. Also,</td></tr><tr><th id="L84"><a href="#L84">84</a></th><td>removing those words helps to make the index smaller and searching faster.</td></tr><tr><th id="L85"><a href="#L85">85</a></th><td></td></tr><tr><th id="L86"><a href="#L86">86</a></th><td>== Known issues and To-do's ==</td></tr><tr><th id="L87"><a href="#L87">87</a></th><td>* Known issue: Relevanssi doesn't play nice with widgets that display recent posts. Right now it makes them disappear. Help with this problem would be most welcome.</td></tr><tr><th id="L88"><a href="#L88">88</a></th><td>* Known issue: In general, multiple Loops on the search page may cause surprising results, and please make sure the actual search results are the first loop.</td></tr><tr><th id="L89"><a href="#L89">89</a></th><td>* Known issue: Relevanssi doesn't necessarily play nice with plugins that modify the excerpt. If you're having problems, try using relevanssi_the_excerpt() instead of the_excerpt().</td></tr><tr><th id="L90"><a href="#L90">90</a></th><td>* To-do: The stop word list management needs small improvements.</td></tr><tr><th id="L91"><a href="#L91">91</a></th><td>* To-do: Improve the display of query logs. Any requests? What information would you like to see, what would be helpful?</td></tr><tr><th id="L92"><a href="#L92">92</a></th><td>* To-do: Category and tag exclusion from search results.</td></tr><tr><th id="L93"><a href="#L93">93</a></th><td></td></tr><tr><th id="L94"><a href="#L94">94</a></th><td>== Changelog ==</td></tr><tr><th id="L95"><a href="#L95">95</a></th><td></td></tr><tr><th id="L96"><a href="#L96">96</a></th><td>= 1.4 =</td></tr><tr><th id="L97"><a href="#L97">97</a></th><td>* Added an option to restrict searches to certain categories or tags, either by plugin option or hidden input field in the search form.</td></tr><tr><th id="L98"><a href="#L98">98</a></th><td>* The contents of `&lt;script&gt;` and other such tags are now removed from excerpts.</td></tr><tr><th id="L99"><a href="#L99">99</a></th><td>* When indexing, HTML tags and `[quicktags]` are removed.</td></tr><tr><th id="L100"><a href="#L100">100</a></th><td>* Digits are no longer removed from terms. Re-index database to get them indexed.</td></tr><tr><th id="L101"><a href="#L101">101</a></th><td>* Wrapped the output of `relevanssi_the_excerpt()` in &lt;p&gt; tags.</td></tr><tr><th id="L102"><a href="#L102">102</a></th><td>* Stopwords are no longer removed from search queries.</td></tr><tr><th id="L103"><a href="#L103">103</a></th><td>* Search result snippet length can now be determined in characters or whole words.</td></tr><tr><th id="L104"><a href="#L104">104</a></th><td></td></tr><tr><th id="L105"><a href="#L105">105</a></th><td>= 1.3.3 =</td></tr><tr><th id="L106"><a href="#L106">106</a></th><td>* Small bug fixes, removed the error message caused by a query that is all stop words.</td></tr><tr><th id="L107"><a href="#L107">107</a></th><td>* Content and excerpt filters are now applied to excerpts created by Relevanssi.</td></tr><tr><th id="L108"><a href="#L108">108</a></th><td>* Default highlight CSS class has a unique name, `search-results` was already used by WordPress.</td></tr><tr><th id="L109"><a href="#L109">109</a></th><td></td></tr><tr><th id="L110"><a href="#L110">110</a></th><td>= 1.3.2 =</td></tr><tr><th id="L111"><a href="#L111">111</a></th><td>* Quicktags are now stripped from custom-created excerpts.</td></tr><tr><th id="L112"><a href="#L112">112</a></th><td>* Added a function `relevanssi_the_excerpt()', which prints out the excerpt without triggering `wp_trim_excerpt()` filters.</td></tr><tr><th id="L113"><a href="#L113">113</a></th><td></td></tr><tr><th id="L114"><a href="#L114">114</a></th><td>= 1.3.1 =</td></tr><tr><th id="L115"><a href="#L115">115</a></th><td>* Another bug fix release.</td></tr><tr><th id="L116"><a href="#L116">116</a></th><td></td></tr><tr><th id="L117"><a href="#L117">117</a></th><td>= 1.3 =</td></tr><tr><th id="L118"><a href="#L118">118</a></th><td>* New query logging feature.
